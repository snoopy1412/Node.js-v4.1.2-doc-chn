关于此文档#

本文档的目标是从参考和概念的角度全面解释 Node.js 的 API来全面的解释Node.js API。每一 章节描述的是一个内置模块或高级概念

在标题下面的列表详细说明了, 属性类型, 函数参数, 以及事件处理器参数

每一个.html文档都有其对应的.json文档，用有结构的方式呈现出相同的信息。 但是这个特性是实验性的， 因为有利于想通过此文档进行程序设计开发的IDE和实用工具，所以增加到了文档中

每一个.html和.json文件通过相应的.markdown文件生成，这些.markdown 文件位于Node.js源代码树的doc/api/文件夹。 文档是通过tools/doc/generate.js程序来生成的，HTML模板的位置位于doc/template.html

Stability Index(稳定度)#

在文档中, 你可以了解每个章节的稳定度。 Node.js API在不断的改变中, 随着它的成熟, 某些部分会变得比其他更加可靠。 一些模块经过严格的验证, 在开发中有很高的依赖度, 所以它们几乎不会改变. 而其他的一些模块是全新的、实验性的或证实有危险性的正在重新设计中

稳定度定义如下: 

Stability: 0 - Deprecated(弃用)
此功能被认定为有问题的, 注定会改变.  不要依赖它们，否则可能会引起警告.
不应该考虑向后兼容
Stability: 1 - Experimental(实验性)
此功能可能会改变, 并且被命令行标记封闭
在未来的版本中可能会被移除或者改变
Stability: 2 - Stable(稳定)
此API被证明是没问题的。 与npm生态系统的兼容性有着很高的优先级，
并且不会被破坏，除非绝对必要。
Stability: 3 - Locked(锁定)
只会因为安全、性能、漏洞修复问题被修改
尝试改变这部分的API将会被拒绝
