Assert(断言)#

Stability: 2 - Stable

此模块用于编写应用中的测试用例。可以通过require('assert')调用

assert.fail(actual, expected, message, operator)#

抛出一个异常，显示用例的实际值(actual)和期望值(expected)，通过分隔符(operator)隔开

assert(value[, message]), assert.ok(value[, message])#

测试结果是否为真。相当于 assert.equal(true, !!value, message).

assert.equal(actual, expected[, message])#

浅测试，等同于使用( == )进行相等判断 .

assert.notEqual(actual, expected[, message])#

浅测试，等同于使用'！='进行不相等判断 ( != ).

assert.deepEqual(actual, expected[, message])#

深度匹配测试。原始值与( == )相比较。 对象原型除外

assert.notDeepEqual(actual, expected[, message])#

非深度匹配测试。 与assert.deepEqual相反

assert.strictEqual(actual, expected[, message])#

严格相等匹配测试，等同用( === )判断匹配

assert.notStrictEqual(actual, expected[, message])#

非严格相等匹配测试，等同于( !== )判断不匹配

assert.deepStrictEqual(actual, expected[, message])#

深度严格匹配测试，原始值与 ( === )相比较

assert.notDeepStrictEqual(actual, expected[, message])#

非深度严格匹配测试. 与 assert.deepStrictEqual相反

assert.throws(block[, error][, message])#

声明一个block用于抛出错误，error可以是构造函数，验证函数或者正则表达式

用构造函数验证实例： 

assert.throws(
    function() {
    throw new Error("Wrong value");
    },
    Error
    );

用正则表达式验证信息错误： 

assert.throws(
    function() {
    throw new Error("Wrong value");
    },
    /value/
    );

自定义错误验证： 

assert.throws(
    function() {
    throw new Error("Wrong value");
    },
    function(err) {
    if ( (err instanceof Error) && /value/.test(err) ) {
    return true;
    }
    },
    "unexpected error"
    );

assert.doesNotThrow(block[, error][, message])#

声明一个block不抛出错误。详见assert.throws()

如果一个block抛出了一个错误并且与error的类型不同, 那么这个抛出的错误将被反向传播到调用方。 在断言中没有匹配到错误类型时，下面这个例子将会抛出TypeError异常 

assert.doesNotThrow(
    function() {
    throw new TypeError("Wrong value");
    },
    SyntaxError
    );

若error匹配到了从block中抛出的错误，那么取而代之将抛出AssertionError

assert.doesNotThrow(
    function() {
    throw new TypeError("Wrong value");
    },
    TypeError
    );

assert.ifError(value)#

当value为真抛出。常用于回调中第一个 error 参数的检查。
